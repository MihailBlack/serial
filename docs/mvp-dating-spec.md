# MVP знакомств на мероприятиях

## Цель
Система помогает участникам мероприятий быстро находить новые полезные контакты без регистрации и сложных форм.

## Основной сценарий
1. Пользователь открывает страницу.
2. Заполняет анкету: ФИО, телефон, краткое описание деятельности.
3. Нажимает "Сохранить анкету".
4. Нажимает кнопку "Кубики" и получает случайного человека для знакомства.

## Сущность `Profile`
```json
{
  "id": "string",
  "fullName": "string",
  "phone": "string",
  "about": "string",
  "createdAt": "ISO-8601 datetime"
}
```

## Правила валидации
- `fullName`: обязательно, от 5 до 120 символов.
- `phone`: обязательно, строка 7-20 символов, может содержать `+`, пробелы, скобки и дефисы.
- `about`: обязательно, от 10 до 300 символов.
- `createdAt`: автоматически ставится сервером или клиентом в ISO-формате.

## Минимальный API-контракт

### `POST /profiles`
Создает новую анкету.

**Request body**
```json
{
  "fullName": "Иванов Иван Иванович",
  "phone": "+7 999 123-45-67",
  "about": "Развиваю B2B SaaS продукты и помогаю командам с ростом."
}
```

**Response 201**
```json
{
  "id": "usr_001",
  "fullName": "Иванов Иван Иванович",
  "phone": "+7 999 123-45-67",
  "about": "Развиваю B2B SaaS продукты и помогаю командам с ростом.",
  "createdAt": "2026-03-26T10:00:00Z"
}
```

### `GET /profiles`
Возвращает список анкет.

### `GET /profiles/random`
Возвращает случайную валидную анкету для кнопки "Кубики".

**Response 200**
```json
{
  "profile": {
    "id": "usr_007",
    "fullName": "Петрова Анна Сергеевна",
    "phone": "+7 921 555-13-42",
    "about": "Организую образовательные события и развиваю комьюнити."
  }
}
```

**Response 404 (если некого показывать)**
```json
{
  "message": "Пока нет участников для знакомства"
}
```

## Логика кнопки "Кубики"
1. Взять все анкеты.
2. Отфильтровать только валидные (`fullName`, `phone`, `about` заполнены).
3. Если список пуст, вернуть сообщение об отсутствии участников.
4. Если анкета одна, вернуть ее.
5. Иначе выбрать индекс через `Math.floor(Math.random() * validProfiles.length)` и вернуть соответствующую анкету.

## Нефункциональные требования MVP
- Работа без регистрации и авторизации.
- Быстрый отклик кнопки "Кубики" (до 300 мс при малом объеме данных).
- Логику рандома держать простой и равновероятной.
